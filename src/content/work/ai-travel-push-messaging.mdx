### _**Re-engineering Travel Engagement with an AI Messaging Orchestrator**_

#### **The Opportunity**

A leading online travel platform operating across Germany, Austria, Switzerland, the Netherlands, and Spain already pushed millions of promotional messages over Whatsapp, Telegram, and other chat channels. But every campaign looked the same to every customer.

They partnered with us to personalize promotional marketing instead of doing generic deal blasts. We helped them move away from a “spray-and-pray” approach that left engagement flat and provided additional opportunity to answer follow-up questions without flooding agents or rebuilding existing systems.

The business objective was clear: **transform messaging into a high-performing sales and support channel** - with a personalized touch.

#### **The Solution: One Orchestrator, Every Offer**

At the core of the solution was a **unified AI Messaging Orchestrator** that sits between the user, the CRM, and the offer-inventory platform. The orchestrator is powered by an LLM and makes function calls to external systems as required. 

1.  **User-profile retrieval** - A function call pulls the latest profile information (demographic details, budget, preferred destinations, previous bookings) when a user message arrives or a campaign is triggered.
    
2.  **Offer-inventory query** - The LLM crafts a second function call to the offer-inventory API, by determining parameters such as destination, travel window, price ceiling, and holiday style that are extracted from the user message and profile.
    
3.  **LLM-driven ranking and filtering** - From dozens of returned offers, the LLM scores and selects the most relevant three to five holiday offers, matching both the user profile and their current ad-hoc query.
    
4.  **Channel-native presentation** - A Whatsapp (or Telegram) API call builds a product-catalog carousel inside the chat, so the user can swipe rather than scroll through the travel offers.
    
5.  **On-Demand RAG for follow-up questions** - If the user taps an offer and asks “Does this include checked luggage?” the orchestrator crawls that specific offer URL in real time, retrieves fresh facts and answers instantly. Given the lower response rates; this solution was preferred instead of a traditional RAG-based approach.
    

#### **Hero Moment: From Query to Conversion in Seconds**

One example stands out.

A customer messaged the travel platform on Telegram:

> "Any sunny places under €600 next week? 😎"

1.  Orchestrator fetches the user profile (user location:Cologne, beach lover, short-haul, prefers 4-star).
    
2.  LLM queries inventory: price ≤ 600, date='15-Dec to 22-Dec', type=beach.
    
3.  47 offers return; the LLM ranks the top three.
    
4.  WhatsApp carousel appears in-chat with images, prices, summary details and link to actual deal information.
    
5.  The customer can select the Mallorca deal and ask questions or clarifications inside the chat thread.
    

No app download, no call center queue. Just travel, reimagined.
