{/* Open with a hook paragraph. No heading — the page template already
    renders the title as h1. Start at h2 (##) for section headings. */}

This is the opening paragraph. It should pull the reader in — a specific
observation, a tension, a number that surprised you. Avoid "In this post
we'll discuss..." preambles; just start with the substance.

{/* ── Section headings — use ## for top-level sections in the body.
       Use ### for subsections, #### sparingly. Don't use # — that's the title. */}

## A section heading (h2)

Body text. **Bold for emphasis** when something genuinely matters.
_Italic for softer emphasis_ — typically titles, foreign phrases, or
introducing a term. Use `inline code` for filenames, command names,
function names, or short technical strings.

### A subsection heading (h3)

More body content.

#### A subsubsection (h4 — use rarely)

Most posts don't need this depth.

{/* ── Links — internal and external. Internal links should be relative
       paths (start with /), not full URLs. */}

You can [link to another post on this site](/blog/some-other-slug)
using a relative path. External links work the same way:
[Astro docs](https://docs.astro.build) opens in the same tab by default.

{/* ── Lists. Unordered uses `-`. Ordered uses `1.` `2.` etc. The
       numbers don't have to be in order — Markdown renumbers them. */}

A bulleted list:

- First item
- Second item, with **bold** inside
- Third item

A numbered list:

1. First step
2. Second step
3. Third step

{/* ── Blockquotes — for pull quotes, citations, or sources. */}
